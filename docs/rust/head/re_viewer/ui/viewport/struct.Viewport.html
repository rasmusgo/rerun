Struct re_viewer::ui::viewport::Viewport

source · 
pub struct Viewport {
    space_views: HashMap<SpaceViewId, SpaceView>,
    visible: BTreeSet<SpaceViewId>,
    trees: HashMap<BTreeSet<SpaceViewId>, Tree<SpaceViewId>>,
    maximized: Option<SpaceViewId>,
    has_been_user_edited: bool,
    space_view_entity_window: Option<SpaceViewEntityPicker>,
}
Expand description

Describes the layout and contents of the Viewport Panel.

Fields§

§space_views: HashMap<SpaceViewId, SpaceView>

Where the space views are stored.

§visible: BTreeSet<SpaceViewId>

Which views are visible.

§trees: HashMap<BTreeSet<SpaceViewId>, Tree<SpaceViewId>>

The layouts of all the space views.

One for each combination of what views are visible. So if a user toggles the visibility of one SpaceView, we switch which layout we are using. This is somewhat hacky.

§maximized: Option<SpaceViewId>

Show one tab as maximized?

§has_been_user_edited: bool

Set to true the first time the user messes around with the viewport blueprint. Before this is set we automatically add new spaces to the viewport when they show up in the data.

§space_view_entity_window: Option<SpaceViewEntityPicker>
